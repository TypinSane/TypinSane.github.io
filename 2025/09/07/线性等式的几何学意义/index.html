记录一些问题
2025-9-8 22:13 昨天还正常的hexo git 插件，今天在deploy的时候就报错， 什么spawn failed，查了很多网上的资料都没有找到能解决的方法， 普遍说是因为请求github的网络问题。但是我对自己深信不疑， 因为毕竟昨天才正常deploy并且成功了，刻板的以为是我在_post文件夹中初始化了一个仓库造成的。 最后到这个目录中手动push了一次， 才确定了是与github的连接有问题。请求ssh -T git@github.com 22端口 timeout, 但是明明昨天还是一样的配置， 最终按照这篇[笔记](坑：ssh: connect to host github.com port 22: Connection refused - 知乎)的指令测试了确实是改用443端口就正常了。 于是对.ssh/config.yml进行了修改。终于解决了这个问题。 有时候陷入了一种固执的思维模式， 答案都告诉我了， 但是自己依然没法迅速的走出来。
